KEY FEATURES IMPLEMENTED:

   - Auto-save on credential updates
   - Survive restarts without QR re-scan
   - Graceful session creation on first run

2. BOT INITIALIZATION (index.ts)
   - Create Baileys makeWASocket with Supabase auth state
   - QR code generation (ASCII terminal output)
   - Connection lifecycle management (connecting → open → close)
   - Auto-reconnect with 5-second delay on disconnect
   - Graceful shutdown handler (SIGTERM/SIGINT)

3. MESSAGE LISTENING (listener.ts)
   - Filter: Skip own messages, non-group messages
   - Register: messages.upsert event on Baileys socket
   - Classify: Text/Image/Document/Irrelevant
   - Validate: Check group is registered (is_active=true)
   - Process: Only "notify" events (skip history sync)
   - Route: Build ClassifiedMessage and call routeMessage()

4. MEDIA DOWNLOAD (media.ts)
   - Handle images, videos, documents
   - Decrypt via Baileys downloadMediaMessage()
   - MIME type detection
   - File size validation (max 10MB)
   - Return: buffer + mimeType + filename
   - Error handling: return null on failure

5. PROCESS MANAGEMENT (start-bot.ts)
   - Dynamic import to avoid loading in Next.js builds
   - Health check loop (60-second interval)
   - Graceful shutdown (clear interval, await shutdownBot())
   - Uncaught exception handler
   - Unhandled rejection handler
   - Structured logging

EXPECTED BEHAVIOR:
1. Bot starts, loads session from Supabase
2. If new session: generates QR code, displays in terminal
3. User scans QR with WhatsApp
4. Bot connects and logs "Bot ready to receive messages"
5. Bot listens for incoming group messages
6. For each message:
   - Check if group is registered
   - Classify message type
   - Download media if needed
   - Route to ai-parser (Phase 3b)
7. On shutdown (SIGTERM): gracefully close socket and exit
